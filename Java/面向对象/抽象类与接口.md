# Day XX - 抽象类（Abstract）与接口（Interface）

---

# 一、抽象方法（abstract）

## 什么是抽象方法

抽象方法只有方法声明，没有具体实现。

语法：

```java
abstract 返回值类型 方法名(参数列表);
```

例如：

```java
abstract void eat();
```

特点：

- 使用 `abstract` 修饰。
- 没有方法体（没有 `{}`）。
- 子类必须重写该方法才能使用。

---

# 二、抽象类（abstract class）

## 什么是抽象类

使用 `abstract` 修饰的类称为抽象类。

语法：

```java
abstract class Animal{
}
```

例如：

```java
public abstract class Animal{
    String name;
    int age;
    String color;

    abstract void eat();

    void drink(){
        ...
    }
}
```

---

## 抽象类特点

### 1、不能实例化

抽象类不能创建对象。

错误示例：

```java
Animal animal = new Animal();
```

必须由子类继承后创建对象：

```java
Dog dog = new Dog();
Fish fish = new Fish();
```

---

### 2、可以包含普通方法

例如：

```java
void drink(){
    ...
}
```

子类可以直接继承使用，无需重写。

---

### 3、可以包含抽象方法

例如：

```java
abstract void eat();
```

子类必须实现。

---

### 4、子类必须重写所有抽象方法

例如：

```java
class Dog extends Animal{
    void eat(){
        ...
    }
}
```

如果没有全部重写：

那么子类也必须声明为抽象类。

```java
abstract class Dog extends Animal{
}
```

一般开发中很少这样使用。

---

# 三、抽象类的意义（老师重点）

抽象类主要有两个作用：

### ① 封装共有属性和行为

例如：

Animal 中统一拥有：

- name
- age
- color
- drink()

所有动物直接继承即可。

实现：

> **代码复用**

---

### ② 定义统一的方法规范

例如：

```java
abstract void eat();
```

所有动物必须拥有 eat()。

但是：

每种动物实现方式不同。

例如：

狗：

```
吃骨头
```

鸡：

```
吃小米
```

鱼：

```
吃小虾
```

因此：

抽象方法负责统一入口。

具体实现交给子类完成。

---

# 四、接口（Interface）

## 什么是接口

接口也是一种引用数据类型。

语法：

```java
interface 接口名{
}
```

例如：

```java
public interface Swim{
    void swim();
}
```

---

## 接口特点

### 1、不能实例化

错误：

```java
Swim s = new Swim();
```

接口只能由类实现。

---

### 2、只能包含抽象方法（当前阶段）

例如：

```java
void swim();
```

老师说明：

目前先学习抽象方法。

默认方法、静态方法等以后再学习。

---

### 3、实现接口必须重写所有方法

例如：

```java
class Dog implements Swim{
    public void swim(){
        ...
    }
}
```

注意：

实现接口的方法必须使用 **public** 修饰。

---

### 4、一个类可以实现多个接口

例如：

```java
class Dog extends Animal implements Swim{
}
```

老师强调：

如果既继承又实现：

> **先 extends，后 implements。**

---

### 5、接口可以继承接口

例如：

```java
interface A extends B{
}
```

---

# 五、抽象类与接口的区别

| 抽象类 | 接口 |
|---------|------|
| abstract class | interface |
| 可以有成员变量 | 当前主要学习抽象方法 |
| 可以有普通方法 | 当前主要学习抽象方法 |
| 可以有抽象方法 | 方法默认都是抽象方法 |
| 使用 extends 继承 | 使用 implements 实现 |
| 一个类只能继承一个抽象类 | 一个类可以实现多个接口 |

# 六、Animal 案例理解

Animal：

负责保存所有动物共有内容：

- name
- age
- color
- drink()

并规定：

```java
eat()
```

必须存在。

---

Dog：

- 重写 eat()
- 新增 lookHome()
- 实现 Swim 接口

---

Fish：

- 重写 eat()
- 实现 Swim 接口

---

Chick：

- 重写 eat()
- 新增 layEggs()

---

这样：

所有动物都有统一的行为入口：

```java
eat();
drink();
```

而各自保留自己的特有功能。

---

# 七、老师重点总结

- abstract 修饰抽象方法和抽象类。
- 抽象方法没有方法体。
- 抽象类不能实例化。
- 子类必须重写所有抽象方法。
- 抽象类用于封装公共代码，实现代码复用。
- 接口用于定义统一规范。
- implements 表示实现接口。
- 实现接口的方法必须使用 public 修饰。
- 一个类可以实现多个接口。
- **继承使用 extends，实现接口使用 implements。**

## 代码

```
/** 接口的演示 */
public class InterfaceDemo {
    public static void main(String[] args) {
        //Inter inter = new Inter();//编译错误，接口不能被实例化
    }
}
//演示接口的定义:
interface Inter{
    public abstract void show();
    void test();//接口中的方法默认都是公开抽象的，前面默认有public abstract
    //void say(){}//编译错误，抽象方法不能有方法体
}
//演示接口的实现
class InterImpl implements Inter{
    public void show(){//重写接口中的抽象方法时，必须加public
    }
    public void test(){
    }
}

//演示接口的多实现
interface Inter1{
    void show();
}
interface Inter2{
    void test();
}
abstract class Eoo{
    abstract void say();
}
class Foo extends Eoo implements Inter1,Inter2{
    public void show(){}
    public void test(){}
    void say(){}
}

//演示接口继承接口
interface Inter3{
    void show();
}
interface Inter4 extends Inter3{
    void test();
}
class Goo implements Inter4{
    public void test(){}
    public void show(){}
}
```
